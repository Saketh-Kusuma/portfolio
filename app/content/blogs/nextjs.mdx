# Building Modern Web Apps with Next.js

Next.js is a powerful React framework that helps developers ship fast, scalable, and SEO-friendly web applications.

In this post, we’ll explore what makes Next.js such a strong choice, how it works, and best practices for building modern sites.

## Why Next.js?

Next.js brings production-ready features to React out of the box:

- **Hybrid rendering**: support for static generation, server rendering, and client-side hydration.
- **File-system routing**: pages and routes are based on files under `app/` or `pages/`.
- **Built-in performance**: automatic code splitting, optimized images, and asset loading.
- **Developer experience**: fast refresh, TypeScript support, and a rich ecosystem.

## Core Concepts

### 1. App Router

The App Router, introduced in Next.js 13, organizes routes using the `app/` directory. Each folder can include `page.tsx`, `layout.tsx`, and `template.tsx` to define UI structure and route behavior.

Example:

```tsx
// app/blog/[slug]/page.tsx
import { notFound } from "next/navigation";
import { blogs } from "../../lib/blogs";

export default async function BlogPage({ params }) {
  const { slug } = params;
  const loader = blogs[slug];

  if (!loader) {
    notFound();
  }

  const Blog = (await loader()).default;
  return <Blog />;
}
```

### 2. Server Components

Next.js uses React Server Components by default in the `app/` directory. Server components can fetch data securely on the server and stream HTML to the client.

### 3. Data Fetching

Next.js provides flexible data fetching options:

- `fetch()` with built-in caching and revalidation
- `getServerSideProps` and `getStaticProps` in the pages router
- `use` with promises in server components

Example with `fetch` in a server component:

```tsx
const res = await fetch("https://api.example.com/data", { cache: "force-cache" });
const data = await res.json();
```

### 4. Static Generation and SSR

Next.js makes it easy to choose the right rendering strategy for each page:

- `static` generation for content that rarely changes
- `dynamic` or `force-dynamic` for content that updates frequently
- `revalidate` for incremental static regeneration

Use `export const dynamic = "auto";` or `export const revalidate = 60;` to control caching behavior.

## Building a Blog with MDX

MDX is a natural fit for content-driven sites. You can render Markdown with React components and create a blog system with file-based sources.

Example structure:

- `app/content/blogs/nextjs.mdx`
- `app/content/blogs/tailwindcss.mdx`
- `app/lib/blogs.ts`
- `app/blog/[slug]/page.tsx`

This enables simple content imports and dynamic blog pages while preserving the benefits of Next.js rendering.

## Performance Best Practices

### Optimize Images

Use the built-in `next/image` component for responsive, lazy-loaded images.

```tsx
import Image from "next/image";

<Image
  src="/images/hero.png"
  alt="Hero"
  width={1200}
  height={800}
  priority
/>
```

### Cache API Requests

Use caching strategies with `fetch()` and edge caching to reduce repeated requests.

```ts
const res = await fetch(process.env.API_URL, {
  next: { revalidate: 300 },
});
```

### Minimize JavaScript

Keep interactive behavior scoped to client components, and render the rest on the server.

```tsx
"use client";

import { useState } from "react";
```

## Real-World Use Cases

Next.js is ideal for:

- marketing websites and landing pages
- SaaS dashboards
- e-commerce stores
- documentation sites
- blogs and content platforms

## Tips for a Smooth Upgrade

If you are migrating from Create React App or the pages router:

1. Convert key views into server components.
2. Use the `app/` directory for new routes.
3. Add `next.config.ts` and enable any required experimental features.
4. Keep components small and reuse layouts with `layout.tsx`.

## Conclusion

Next.js offers a modern, flexible stack for building web applications with React. It combines fast rendering, strong SEO, and a great developer experience.

For most projects, Next.js is a reliable choice when you want to move quickly without sacrificing performance.
